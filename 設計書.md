# YouTube ダウンロードアプリ 設計書（Desktop / WPF 版）

## 1. アプリ概要

### 1.1 目的
Windows PC 上で、ユーザーが指定した YouTube 動画／プレイリストを yt-dlp および ffmpeg を用いて
ダウンロード・変換・整理するための GUI デスクトップアプリケーションを提供する。

### 1.2 前提技術
- OS: Windows 10 / 11
- 言語・フレームワーク:
  - .NET 8 以降
  - WPF (Windows Presentation Foundation)
  - MVVM アーキテクチャ
- 外部ツール:
  - yt-dlp (外部実行ファイル)
  - ffmpeg (外部実行ファイル)
- 永続化:
  - 設定: JSON ファイル
  - メタデータ: v1.0 では JSON / 必要に応じて SQLite に移行可能な設計

## 2. 要求仕様

### 2.1 機能要件

#### F-01: 動画 URL 入力・解析
- ユーザーは URL 入力テキストボックスに YouTube URL を入力できる。
- 「解析」ボタン押下で、URL が単一動画かプレイリストかを判定する。
- yt-dlp (`--dump-json` など) を用いてメタデータを取得し、以下をプレビュー表示する：
  - サムネイル
  - タイトル
  - チャンネル名
  - 種別（単一動画／プレイリスト）
  - 動画本数（プレイリスト）

#### F-02: プレイリスト対応
- プレイリスト URL の場合、プレイリスト全体のメタ情報を取得する。
- 各動画の一覧（タイトル／長さ／公開日等）を DataGrid 等で表示する。
- プレイリスト名を元にフォルダを自動生成し、その中に動画ファイルを保存する。
- 「すべてダウンロード」「選択した動画のみダウンロード」を選択可能。

#### F-03: ダウンロード設定
- 保存先フォルダをフォルダブラウズダイアログで指定可能（デフォルト: ユーザーのビデオフォルダ配下）。
- ダウンロード種別：
  - 動画＋音声（mp4, mkv 等）
  - 音声のみ（mp3, m4a 等）
- 解像度／品質の選択：
  - 例: best / 1080p / 720p / 音声ビットレート 等
- ファイル名テンプレート：
  - 例: `{index:02d} - {title}`, `{title} - {channel}` など
  - 使用可能プレースホルダは設定画面に説明を表示する。

#### F-04: ダウンロード実行・進捗表示
- ダウンロードは BackgroundWorker, Task, async/await 等で UI スレッドと分離して実行する。
- ダウンロードキューを表示する ListView / DataGrid を用意し、各ジョブごとに：
  - タイトル
  - 状態（待機中／実行中／完了／失敗）
  - 進捗バー（0〜100%）
  - 残り時間（任意）
- ユーザーは個々のジョブをキャンセル可能。

#### F-05: メタデータ取得・保存
- yt-dlp の JSON 出力から以下の情報を抽出する：
  - 動画 ID
  - タイトル
  - チャンネル名
  - 再生時間
  - 公開日
  - サムネイル URL
  - プレイリスト ID / プレイリストタイトル（該当時）
- ダウンロード完了時に、メタデータを更新しローカルに保存する。
  - v1.0: 単一 JSON もしくは複数 JSON ファイル
  - 将来: SQLite DB による管理に移行可能
- 「履歴／ライブラリ」画面でメタデータを一覧表示し、検索・ソートを可能にする。

#### F-06: ffmpeg による変換・メタタグ埋め込み
- yt-dlp で取得したファイルを、ffmpeg 経由で指定フォーマットに変換する。
- 音声のみ抽出モードでは、音声トラックを mp3 / m4a 等に変換する。
- 可能な範囲で以下のタグを埋め込む：
  - Title: 動画タイトル
  - Artist: チャンネル名
  - Album: プレイリスト名（任意）

#### F-07: エラー処理・リトライ
- 代表的なエラー種別を分けてユーザーに表示する：
  - ネットワークエラー
  - URL 不正
  - 解析失敗
  - yt-dlp / ffmpeg 実行ファイル未設定
- ジョブ単位で「リトライ」操作を提供する。
- 必要に応じてエラーログをテキストファイルに出力する。

#### F-08: 設定管理
- 以下の設定項目を持つ：
  - yt-dlp 実行ファイルパス
  - ffmpeg 実行ファイルパス
  - デフォルト保存先フォルダ
  - デフォルトフォーマット（動画／音声）
  - デフォルト品質
  - ファイル名テンプレート
- 設定は JSON ファイルとしてユーザー毎のプロファイルディレクトリ配下に保存する。

## 3. 非機能要件

### 3.1 パフォーマンス
- プレイリスト解析（〜100本程度）は 3〜15秒程度を目安とする。
- ダウンロードの同時実行数は 1〜3 件程度に制限し、過負荷を防止する。

### 3.2 UI / UX
- レイアウト:
  - 左側: ナビゲーション（ダウンロード / ライブラリ / 設定）
  - 右側: メインコンテンツ
- ダークテーマ / ライトテーマの切り替えは将来対応とし、リソース辞書で拡張可能にする。
- 操作フローは以下を基本とする：
  1. URL 入力
  2. 解析
  3. 設定確認
  4. ダウンロード開始

### 3.3 配布
- 配布形態は以下を想定:
  - 単一 EXE（self-contained）または MSIX インストーラー
- yt-dlp / ffmpeg の扱い:
  - デフォルト: ユーザーにパスを指定させる（ダウンロードユーザーの自己責任）。
  - オプション: 同梱する場合はライセンス条件を別途確認。

## 6. 処理フロー

### 6.1 URL 解析フロー

1. ユーザーが DownloadView で URL を入力し、`AnalyzeCommand` を実行。
2. DownloadViewModel が `YtDlpClient.AnalyzeUrlAsync` を呼び出す。
3. YtDlpClient が yt-dlp プロセスを `--dump-json` で起動し、JSON を取得。
4. JSON を `VideoMetadata` / `PlaylistMetadata` にマッピング。
5. DownloadViewModel がプロパティを更新し、ビューにプレビューを表示。
6. プレイリストの場合、ユーザーが「詳細表示」→ PlaylistSelectionDialog を開く。

### 6.2 プレイリストダウンロードフロー

1. PlaylistSelectionDialog で動画の選択を行い「OK」。
2. DownloadViewModel が選択された動画ごとに `DownloadJob` を生成。
3. プレイリスト名から保存フォルダ (`{root}/{playlist_title}/`) を決定。
4. `DownloadManager.Enqueue` にジョブを登録。
5. DownloadManager がバックグラウンドで順次 `YtDlpClient.DownloadAsync` を呼び出す。
6. 必要に応じてダウンロード後に `FfmpegClient.ConvertAsync` を実行。
7. 完了後、`VideoMetadata` を更新し `MetadataRepository.SaveVideoMetadata` を呼び出す。
8. 進捗・状態はイベントを通じて ViewModel に通知され、UI を更新。

### 6.3 単一動画ダウンロードフロー

1. URL 解析の結果が単一動画。
2. ユーザーが設定を確認し「ダウンロード開始」。
3. DownloadViewModel が 1 件の `DownloadJob` を作成し `DownloadManager.Enqueue`。
4. 以降はプレイリストの各動画と同様。
